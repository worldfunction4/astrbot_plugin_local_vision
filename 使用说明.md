# AstrBot 本地视觉识图插件 - 小白使用指南

## 📖 这是什么？

这是一个让你的 QQ 机器人能够"看懂图片"的插件。

**工作原理**：
- 你的**家用电脑**（有显卡）负责识别图片（算力中心）
- 你的**云服务器**（运行 AstrBot）负责接收消息和发送回复（消息网关）
- 两者通过 Tailscale VPN 安全连接

**为什么要这样设计？**
- 云服务器配置低（2GB 内存）跑不动 AI 模型
- 家用电脑有显卡，但没有公网 IP，不能直接当服务器
- 这个插件把两者的优势结合起来：云端收发消息，本地跑 AI

## ✨ 能做什么？

- 识别图片内容（风景、人物、物品等）
- 识别图片中的文字（OCR）
- 识别动漫角色
- 群聊中 @ 机器人 + 发图 → 自动识别
- 私聊直接发图 → 自动识别

## 📋 支持的平台

- ✅ **QQ**（aiocqhttp/NapCat）：完全支持
- 其它平台目前暂不支持

## 🚀 开始配置（分两步）

### 第一步：配置你的家用电脑（算力中心）

这一步在**你的 Windows/Mac/Linux 电脑**上操作，需要有显卡（NVIDIA/AMD/集成显卡都可以）。

---

#### 1.1 安装 Ollama（AI 模型运行环境）

**Ollama 是什么？**  
就像 Python 需要安装解释器才能运行代码，AI 模型也需要 Ollama 才能运行。

**下载地址**：https://ollama.com/download

- Windows：下载 `.exe` 安装包，双击安装
- Mac：下载 `.dmg` 安装包，拖到应用程序
- Linux：运行官网提供的安装命令

**验证安装**：  
打开命令行（Windows 按 `Win+R` 输入 `cmd`），输入：
```bash
ollama --version
```
如果显示版本号（如 `ollama version is 0.x.x`），说明安装成功。

---

#### 1.2 下载 AI 视觉模型

**推荐模型**（按显存需求排序）：

| 模型名称 | 显存需求 | 特点 | 下载命令 |
|---------|---------|------|---------|
| `llava-phi3` | 4GB | 最快，适合集成显卡 | `ollama pull llava-phi3` |
| `llava:7b` | 8GB | 均衡，速度和效果平衡，但测试下来娱乐性一般 | `ollama pull llava:7b` |
| `llava:13b` | 16GB | 效果更好，需要独立显卡 | `ollama pull llava:13b` |
| `qwen2-vl:7b` | 8GB | 阿里出品，中文更好 | `ollama pull qwen2-vl:7b` |
| `minicpm-v` | 8GB | 适合中等显存，效果不错 | `ollama pull minicpm-v` |

这里我是8GB显存，选择了 `minicpm-v`。

**下载方法**：  
打开命令行，输入（以 `minicpm-v` 为例）：
```bash
ollama pull minicpm-v
```
等待下载完成（大约 4-8GB，需要几分钟到十几分钟）。

**验证下载**：
```bash
ollama list
```
如果看到 `minicpm-v` 出现在列表里，说明下载成功。

---

#### 1.3 安装识图服务的依赖

**这一步是什么？**  
安装一些 Python 库，让识图服务能够运行。

**操作步骤**：

1. 确保你的电脑已安装 Python（3.8 或更高版本）
   - 没有的话去 https://www.python.org/downloads/ 下载安装
   - 安装时勾选 "Add Python to PATH"

2. 下载本插件的 `vision_server.py` 文件到你的电脑（任意文件夹）

3. 打开命令行，进入 `vision_server.py` 所在的文件夹：
   ```bash
   cd C:\你的文件夹路径
   ```

4. 安装依赖：
   ```bash
   pip install fastapi uvicorn httpx
   ```

---

#### 1.4 启动识图服务

**操作步骤**：

在 `vision_server.py` 所在的文件夹，打开命令行，输入：
```bash
python vision_server.py
```

**看到以下输出说明成功**：
```
INFO:     Started server process
INFO:     Uvicorn running on http://0.0.0.0:8765
```

**重要提示**：
- 这个窗口**不要关闭**，关闭了服务就停了
- 想后台运行的话：Windows 用 `pythonw vision_server.py`，Linux 用 `nohup python vision_server.py &`

---

#### 1.5 安装 Tailscale（让云服务器能访问你的电脑）

**Tailscale 是什么？**  
一个虚拟局域网工具，让你的云服务器和家用电脑像在同一个局域网一样互相访问，不需要公网 IP 和端口转发。

**下载地址**：https://tailscale.com/download

- Windows/Mac：下载安装包，双击安装
- Linux：按官网命令安装

**配置步骤**：

1. 安装后打开 Tailscale，登录（可以用 Google/GitHub 账号）
2. 登录成功后，Tailscale 会自动分配一个 IP 地址（如 `100.64.1.2`）
3. 查看你的 Tailscale IP：
   ```bash
   # Windows
   tailscale ip -4
   
   # Mac/Linux
   tailscale ip
   ```
   记下这个 IP，后面要用（例如 `100.64.1.2`）

---

### 第二步：配置你的云服务器（AstrBot）

这一步在**你的云服务器**上操作（运行 AstrBot 的那台服务器）。

---

#### 2.1 在云服务器上安装 Tailscale

**操作步骤**：

1. SSH 登录你的云服务器
2. 安装 Tailscale（Linux）：
   ```bash
   curl -fsSL https://tailscale.com/install.sh | sh
   ```
3. 启动并登录：
   ```bash
   sudo tailscale up
   ```
   会显示一个链接，复制到浏览器打开，用**同一个账号**登录（和家用电脑用的账号一样）

4. 验证连接：
   ```bash
   ping 100.64.1.2
   ```
   （把 `100.64.1.2` 换成你家用电脑的 Tailscale IP）
   
   如果能 ping 通，说明两台机器已经连上了。

---

#### 2.2 测试识图服务是否可访问

在云服务器上运行：
```bash
curl http://100.64.1.2:8765
```
（把 `100.64.1.2` 换成你家用电脑的 Tailscale IP）

如果返回 `{"detail":"Not Found"}`，说明服务可以访问（这是正常的，因为我们没有访问正确的路径）。

---

#### 2.3 安装插件

1. 在 AstrBot 控制台，进入"插件市场"
2. 搜索 `astrbot_plugin_local_vision` 并安装
3. 或者手动下载本插件，放到 `data/plugins/` 目录

---

#### 2.4 配置插件

在 AstrBot 控制台，找到"本地视觉识图"插件，点击"配置"：

| 配置项 | 填什么 | 示例 |
|--------|--------|------|
| `local_vision_api_url` | `http://你的家用电脑Tailscale IP:8765/analyze` | `http://100.64.1.2:8765/analyze` |
| `ollama_model` | 你下载的模型名称 | `minicpm-v` 或 `minicpm-v` 等 |
| `group_require_at` | 建议选 `true`（群聊需要 @ 才触发） | `true` |
| `enable_auto_reply` | 选 `true` | `true` |

其他配置保持默认即可。

**保存配置**，重启 AstrBot。

---

## 🎯 开始使用

配置完成后，就可以开始用了！

### 私聊使用

直接给机器人发送图片，它会自动识别并回复。

**示例**：
```
你：[发送一张猫咪照片]
机器人：这是一只橘色的猫咪，正趴在沙发上，看起来很慵懒...
```

---

### 群聊使用（推荐配置：需要 @）

**直接发图 + @**
```
你：[发送图片] @机器人
机器人：[引用你的消息] 这张图片展示了...
```

**注意**：
- ✅ 发图 + @ 机器人 → 会识别
- ❌ 只发图不 @ → 不会识别（避免误触发表情包和斗图）

---

### 群聊使用（关闭 @ 要求）

如果你在插件配置中把 `group_require_at` 改成 `false`，那么：
- 群里任何人发图 → 机器人都会自动识别

**不推荐这样设置**，因为会导致：
- 表情包被识别
- 斗图时疯狂触发
- 浪费算力

---

## 💡 使用技巧

### 切换模型

想换个更好的模型？

1. 在家用电脑下载新模型：
   ```bash
   ollama pull <模型名称>
   ```

2. 在 AstrBot 插件配置中，把 `ollama_model` 改成 想换的模型名称（如 `minicpm-v`）

3. 保存配置，重启 AstrBot

---

### 自定义识图提示词

想让机器人识图时更关注某些内容？

在插件配置中修改 `custom_prompt`：

**示例 1：只识别文字**
```
请提取图片中的所有文字内容，不要描述图片。
```

**示例 2：识别动漫角色**
```
这是一张动漫图片，请描述角色的外貌、服装、表情和动作。
```

**示例 3：简洁风格**
```
用一句话概括这张图片的主要内容。
```

## 🔧 常见问题

### 问题 1：机器人没反应

**检查清单**：

1. 家用电脑的识图服务是否在运行？
   - 看看运行 `python vision_server.py` 的窗口是否还在
   - 如果关闭了，重新运行

2. Tailscale 是否连接？
   - 在云服务器上运行 `ping 你的家用电脑IP`
   - 如果 ping 不通，检查两台机器的 Tailscale 是否都在线

3. 插件配置的 IP 地址是否正确？
   - 在 AstrBot 插件配置中检查 `local_vision_api_url`
   - 确保 IP 是家用电脑的 Tailscale IP

4. 群聊是否需要 @？
   - 检查 `group_require_at` 是否为 `true`
   - 如果是，记得发图时 @ 机器人

---

### 问题 2：提示"模型未找到"

**错误示例**：
```
❌ 识图失败：本地未找到模型 minicpm-v
请在运行 vision_server 的机器上执行：
ollama pull minicpm-v
```

**解决方法**：

1. 在家用电脑打开命令行，查看已安装的模型：
   ```bash
   ollama list
   ```

2. 如果没有你配置的模型，下载它：
   ```bash
   ollama pull minicpm-v
   ```

3. 或者在 AstrBot 插件配置中，把 `ollama_model` 改成你已经下载的模型名称

---

### 问题 3：识别速度很慢

**可能原因**：
- 显存不足，模型运行缓慢
- 图片太大
- 第一次识图需要加载模型（10-15 秒）

**优化建议**：

1. 换个更小的模型：
   ```bash
   ollama pull llava-phi3
   ```
   然后在插件配置中改成 `llava-phi3`

2. 检查显卡是否被占用：
   - NVIDIA 显卡：运行 `nvidia-smi` 查看显存占用
   - 如果显存被其他程序占满，关闭它们

3. 第一次识图慢是正常的（模型加载需要时间），后续会快很多

---

### 问题 4：提示"无法连接到识图服务"

**错误示例**：
```
❌ 无法连接到识图服务，请确认 vision_server.py 已在本地启动
```

**解决方法**：

1. 检查家用电脑的识图服务是否在运行
2. 在云服务器上测试连接：
   ```bash
   curl http://你的家用电脑TailscaleIP:8765
   ```
3. 如果返回 `Connection refused`，说明服务没启动或 IP 不对
4. 如果返回 `{"detail":"Not Found"}`，说明服务正常（这是正常的响应）

---

### 问题 5：Tailscale 连接不上

**症状**：
- 云服务器 ping 不通家用电脑
- 提示 `Destination Host Unreachable`

**解决方法**：

1. 检查两台机器的 Tailscale 是否都在线：
   ```bash
   tailscale status
   ```

2. 确保两台机器登录的是**同一个 Tailscale 账号**

3. 检查防火墙是否拦截了 Tailscale：
   - Windows：在"Windows 安全中心"→"防火墙"中允许 Tailscale
   - Linux：运行 `sudo ufw allow in on tailscale0`

4. 重启 Tailscale：
   ```bash
   # Windows：在任务栏右键 Tailscale 图标，选择"退出"再重新打开
   # Linux：
   sudo systemctl restart tailscaled
   ```

---

### 问题 6：图片太大无法识别

**错误示例**：
```
图片太大了(15.2MB)，我处理不了...
```

**解决方法**：

在插件配置中增大 `max_image_size_mb`（默认 10MB）：
- 改成 `20` 或 `30`
- 但注意：图片越大，识别越慢

---

## 📊 性能参考

基于 RTX 4060 8GB 显存测试（不一定准确）：

| 模型 | 首次加载 | 识图速度 | 显存占用 |
|------|---------|---------|---------|
| `llava-phi3` | 5秒 | 3-5秒 | 3GB |
| `llava:7b` | 10秒 | 5-8秒 | 6GB |
| `llava:13b` | 15秒 | 10-15秒 | 12GB |
| `qwen2-vl:7b` | 10秒 | 6-9秒 | 7GB |

**说明**：
- 首次识图会加载模型到显存，需要较长时间
- 后续识图速度会快很多
- 实际性能取决于你的显卡
