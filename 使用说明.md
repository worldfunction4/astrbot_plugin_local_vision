# AstrBot 本地视觉识图插件 - 使用说明书

## 📖 简介

这是一个为 AstrBot 开发的本地图片识别插件，通过 Ollama 运行视觉模型（如 LLaVA）来识别图片内容。所有计算都在本地完成，无需调用云端 API，保护隐私且免费使用。

## ✨ 核心特性

- **本地运行** - 所有识图在本地完成，数据不上传云端
- **动态模型切换** - 在 AstrBot 仪表盘随时切换不同的视觉模型
- **智能触发控制** - 群聊可设置需要 @ 才触发，避免误识别表情包
- **自动模型检测** - 发图时自动检查模型是否存在，给出友好提示
- **引用回复** - 机器人回复会引用你的消息，对话更清晰

## 🚀 快速开始

### 1. 安装 Ollama

前往 [Ollama 官网](https://ollama.com/download) 下载并安装 Ollama。

安装完成后，在命令行验证：
```bash
ollama --version
```

### 2. 下载视觉模型

推荐模型（按显存需求排序）：

| 模型 | 显存需求 | 特点 | 下载命令 |
|------|---------|------|---------|
| llava:7b | ~8GB | 平衡性能和速度，推荐 | `ollama pull llava:7b` |
| llava:13b | ~16GB | 更高准确度 | `ollama pull llava:13b` |
| llava:34b | ~32GB | 最高准确度 | `ollama pull llava:34b` |
| llava-phi3 | ~4GB | 轻量级，速度快 | `ollama pull llava-phi3` |

如果不想要死板的也可以换成阿里的qwen系列模型，命令类似。

**首次使用推荐**：
```bash
ollama pull llava:7b
```

### 3. 安装插件依赖

进入插件目录：
```bash
cd 你的路径/astrbot_plugin_local_vision
```

安装 Python 依赖：
```bash
pip install -r requirements.txt
```

### 4. 启动视觉服务

在插件目录运行：
```bash
python vision_server.py
```

看到以下输出表示启动成功：
```
INFO:     Started server process
INFO:     Uvicorn running on http://0.0.0.0:8765
```

**重要**：视觉服务需要一直运行，建议使用以下方式：
- Windows: 使用 `pythonw vision_server.py` 后台运行
- Linux: 使用 `nohup python vision_server.py &` 或配置 systemd 服务
- 或者使用 screen/tmux 保持会话

### 5. 配置插件

在 AstrBot 仪表盘中找到"本地视觉识图"插件，配置以下参数：

| 配置项 | 说明 | 默认值 | 推荐设置 |
|--------|------|--------|---------|
| `vision_server_url` | 视觉服务地址 | `http://127.0.0.1:8765` | 本地使用保持默认 |
| `ollama_model` | 使用的模型名称 | `llava:7b` | 根据显存选择 |
| `default_prompt` | 默认识图提示词 | 请详细描述... | 可自定义 |
| `group_require_at` | 群聊是否需要@才触发 | `true` | 建议开启 |

### 6. 开始使用

配置完成后，在 QQ/微信等平台：

**私聊场景**：
- 直接发送图片 → 机器人自动识别

**群聊场景**（`group_require_at=true`）：
- 发送图片 + @ 机器人 → 触发识别
- 只发图片不 @ → 不触发（避免误识别表情包）

**群聊场景**（`group_require_at=false`）：
- 发送任何图片 → 自动识别

## ⚙️ 高级配置

### 切换模型

1. 在 AstrBot 仪表盘找到插件配置
2. 修改 `ollama_model` 为目标模型名（如 `llava:13b`）
3. 保存配置
4. 确保已通过 `ollama pull` 下载该模型

### 自定义提示词

修改 `default_prompt` 可以改变识图风格：

**详细描述风格**：
```
请详细描述这张图片的内容，包括人物、物品、场景、颜色、氛围等所有细节。
```

**简洁风格**：
```
用一句话概括这张图片的主要内容。
```

**特定领域**：
```
这是一张动漫图片，请描述角色的外貌、服装、表情和动作。
```

### 群聊触发控制

**推荐设置**：`group_require_at = true`
- 优点：避免误识别表情包和斗图，节省算力
- 缺点：每次需要 @ 机器人

**可选设置**：`group_require_at = false`
- 优点：发图即识别，更便捷
- 缺点：群里所有图片都会触发识别

## 🔧 故障排查

### 问题 1：机器人没有反应

**检查清单**：
1. 视觉服务是否在运行？
   ```bash
   curl http://127.0.0.1:8765
   ```
2. 插件是否已启用？（在 AstrBot 仪表盘检查）
3. 群聊是否需要 @？（检查 `group_require_at` 配置）

### 问题 2：提示"模型未找到"

**解决方法**：
```bash
# 查看已安装的模型
ollama list

# 安装缺失的模型
ollama pull llava:7b
```

### 问题 3：识别速度很慢

**可能原因**：
- 显存不足，模型运行缓慢
- 图片尺寸过大

**优化建议**：
1. 切换到更小的模型（如 `llava-phi3`）
2. 确保 Ollama 使用 GPU 加速
3. 检查显存占用：`nvidia-smi`（NVIDIA 显卡）

### 问题 4：视觉服务启动失败

**检查端口占用**：
```bash
# Windows
netstat -ano | findstr :8765

# Linux
lsof -i :8765
```

如果端口被占用，可以修改 `vision_server.py` 最后一行的端口号，同时更新插件配置中的 `vision_server_url`。

### 问题 5：Ollama 连接失败

**检查 Ollama 服务**：
```bash
# 查看 Ollama 是否运行
ollama list

# 如果未运行，启动 Ollama 服务
# Windows: 从开始菜单启动 Ollama
# Linux: systemctl start ollama
```

## 📊 性能参考

基于 RTX 4060 8GB 显存测试：

| 模型 | 首次加载时间 | 单张识图时间 | 显存占用 |
|------|------------|------------|---------|
| llava:7b | ~10秒 | ~5-8秒 | ~6GB |
| llava-phi3 | ~5秒 | ~3-5秒 | ~3GB |
| llava:13b | ~15秒 | ~10-15秒 | ~12GB |

**注意**：
- 首次识图会加载模型到显存，需要较长时间
- 后续识图速度会明显加快
- 实际性能取决于硬件配置和图片复杂度

## 🔒 隐私说明

- ✅ 所有图片识别在本地完成
