# 本地识图功能插件

AstrBot 本地图片识别插件。通过 HTTP 调用本地 Ollama 视觉模型进行图片识别，支持动态切换模型、智能触发控制和引用回复。

支持 QQ（aiocqhttp/NapCat）和微信公众号（weixin_oc）平台。

## 与Astrbot内置的区别
虽然 AstrBot 官方提供了 Ollama 提供商接入，但本插件采用了 “云端网关 + 本地算力中心” 的分布式架构，专门针对特定硬件环境进行了深度优化，服务器配置不够好的也许可以考虑一下，如果配置足够使用内置的即可。

## 选择本插件可以？
1.为“贫民窟”云服务器而生：
如果你使用的是像 2GB 内存 这样的入门级云服务器，运行官方重型视觉 Provider 可能会导致内存抖动甚至进程崩溃。本插件将所有压力转移至本地，云端只负责“传话”。当然缺点就是比较麻烦

2.更懂社交平台的逻辑：
在群聊中，并非所有图片都需要 AI 介入。本插件实现的 “引用回复”识图 功能，允许用户在群友发图后，通过长按回复并 @机器人 的方式唤醒 AI。这不仅保护了你本地 8GB 显存 的带宽，也维护了群聊的秩序。

3.分布式部署的最佳实践：
利用 Tailscale 等内网穿透技术，将云端的 AstrBot 与家中的 Ollama (LLaVA/Qwen-VL) 完美缝合，实现了一套高性能、低成本的私有化 AI 视觉方案。

## 特性

- ✅ **动态模型切换**：在 AstrBot 仪表盘直接切换 Ollama 模型（llava:7b / llava:13b / llava-llama3 等）
- ✅ **智能触发控制**：群聊中可配置为"需要 @ 机器人才触发"，避免误识别表情包和斗图
- ✅ **引用回复支持**：机器人回复时保持引用关系，群聊对话更清晰
- ✅ **模型检测**：自动检测模型是否已安装，给出友好的错误提示
- ✅ **跨平台部署**：AstrBot 运行在服务器，Ollama 运行在本地电脑，通过 Tailscale VPN 连接

## 架构

```
用户发送图片（私聊 / 群聊 @ 机器人）
    │
    ▼
AstrBot 插件接收消息
    │
    ▼
提取图片 URL / 本地路径
    │
    ▼
下载 / 读取图片数据
    │
    ▼
POST 到本地识图服务 (HTTP + Tailscale VPN)
    │
    ▼
识图服务检测模型是否存在
    │
    ▼
调用 Ollama 指定模型进行识别
    │
    ▼
返回描述文本，插件以引用回复方式返回
```

## 依赖

### AstrBot 侧（插件本身）

- `aiohttp >= 3.9.0`（AstrBot 环境通常已自带）

### 本地电脑侧（识图服务）

1. **Ollama**：https://ollama.com/download
2. **llava 视觉模型**（任选其一或多个）：
   ```bash
   ollama pull llava:7b        # 推荐，速度快，显存占用低
   ollama pull llava:13b       # 效果更好，需要更多显存
   ollama pull llava-llama3    # 基于 Llama 3 的版本
   ```
3. **识图服务脚本**：见本仓库 `vision_server.py`

## 部署步骤

### 1. 在本地电脑启动识图服务

本仓库已包含 `vision_server.py`，直接运行即可：

```bash
pip install fastapi uvicorn httpx
python vision_server.py
```

服务将在 `0.0.0.0:8765` 启动，支持以下功能：
- 接受 `model` 参数动态指定 Ollama 模型
- 自动检测模型是否已安装
- 返回友好的错误提示

### 2. 通过 Tailscale / 内网穿透让服务器可访问本地电脑

推荐使用 [Tailscale](https://tailscale.com/) 组建私有网络：

- 本地电脑和服务器都安装 Tailscale 并登录同一账号
- 服务器可通过 Tailscale IP（如 `100.x.x.x:8765`）访问本地识图服务

### 3. 安装插件并配置

在 AstrBot 控制台安装本插件，在插件配置页面设置：

- **`local_vision_api_url`**：识图服务地址（如 `http://100.116.59.47:8765/analyze`）
- **`ollama_model`**：要使用的模型名称（如 `llava:7b`、`llava:13b`）
- **`group_require_at`**：群聊是否需要 @ 才触发（推荐开启）

## 插件配置项

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `local_vision_api_url` | string | `http://100.64.0.1:8765/analyze` | 识图服务地址 |
| `ollama_model` | string | `llava:7b` | Ollama 模型名称（支持动态切换） |
| `enable_auto_reply` | bool | `true` | 是否自动识别图片并回复 |
| `group_require_at` | bool | `true` | 群聊中是否需要 @ 机器人才触发（推荐开启） |
| `reply_prefix` | string | `""` | 回复内容的前缀（可选） |
| `timeout_seconds` | int | `30` | 识图服务超时时间（秒） |
| `max_image_size_mb` | int | `10` | 最大处理图片大小（MB） |
| `custom_prompt` | string | `""` | 自定义识图提示词（留空使用默认） |

## 使用场景

### 私聊
- 直接发送图片 → 自动识别并回复

### 群聊（默认配置：`group_require_at = true`）
- ✅ 发送图片 + @ 机器人 → 触发识别
- ✅ 回复某条消息（带图）+ @ 机器人 → 触发识别，并以引用回复返回结果
- ❌ 只发图片不 @ → 不触发（避免误识别表情包、斗图）

### 群聊（关闭 `group_require_at`）
- 所有图片消息都会被自动识别（不推荐，容易误触发）

## 错误提示

插件会自动检测常见问题并给出友好提示：

- **模型未安装**：`❌ 识图失败：本地未找到模型 llava:13b，请在运行 vision_server 的机器上执行：ollama pull llava:13b`
- **服务未启动**：`❌ 无法连接到识图服务，请确认 vision_server.py 已在本地启动`
- **图片过大**：`图片太大了(15.2MB)，我处理不了...`

## 平台兼容说明

- **QQ（aiocqhttp）**：图片通过 URL 下载，插件自动处理
- **微信公众号（weixin_oc）**：图片保存在容器临时目录，通过本地路径读取
