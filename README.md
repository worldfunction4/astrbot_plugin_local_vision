# AstrBot 本地视觉识图插件

让你的 QQ/微信机器人能够”看懂图片”的插件。

## 💡 核心理念

**云端网关 + 本地算力** 的分布式架构：
- 🌐 **云服务器**（2GB 内存）：运行 AstrBot，负责消息收发
- 💻 **家用电脑**（有显卡）：运行 Ollama 视觉模型，负责图片识别
- 🔒 **Tailscale VPN**：安全连接两者，无需公网 IP

## ✨ 为什么选择这个插件？

虽然 AstrBot 内置了 Ollama 接入，但本插件专为以下场景设计：

### 1. 贫民窟云服务器救星
入门级云服务器（2GB 内存）跑不动 AI 模型？本插件把算力需求转移到你的家用电脑，云端只负责”传话”。

### 2. 更懂社交场景
- ✅ 群聊 @ 触发：避免误识别表情包和斗图
- ✅ 引用回复：机器人回复时保持引用关系，对话更清晰
- ✅ 智能控制：私聊直接识别，群聊需要 @ 才触发

### 3. 分布式部署最佳实践
利用 Tailscale 内网穿透，将云端 AstrBot 与家中 Ollama 完美缝合，实现高性能、低成本的私有化 AI 视觉方案。

## 🚀 快速开始

### 支持的平台
- QQ（aiocqhttp/NapCat）
- 微信公众号（weixin_oc）

### 推荐模型

| 模型 | 显存需求 | 特点 |
|------|---------|------|
| `llava-phi3` | 4GB | 最快，适合集成显卡 |
| `llava:7b` | 8GB | **推荐**，速度和效果平衡 |
| `qwen2-vl:7b` | 8GB | 阿里出品，中文识别更准 |
| `llava:13b` | 16GB | 效果更好，需要独立显卡 |

### 部署架构

```
用户发图 → AstrBot (云服务器) → Tailscale VPN → Ollama (家用电脑) → 返回识别结果
```

## ⚙️ 配置项

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `local_vision_api_url` | 识图服务地址 | `http://100.64.0.1:8765/analyze` |
| `ollama_model` | 模型名称 | `llava:7b` |
| `group_require_at` | 群聊需要 @ 才触发 | `true` |
| `enable_auto_reply` | 自动识别并回复 | `true` |
| `custom_prompt` | 自定义识图提示词 | `””` |
| `max_image_size_mb` | 最大图片大小 | `10` |
| `timeout_seconds` | 超时时间 | `30` |

## 🔧 故障排查

### 机器人没反应？
1. 检查家用电脑的 `vision_server.py` 是否在运行
2. 检查 Tailscale 是否连接（`ping 家用电脑IP`）
3. 群聊是否需要 @？

### 提示”模型未找到”？
```bash
# 查看已安装的模型
ollama list

# 下载模型
ollama pull llava:7b
```

### 识别速度慢？
- 第一次识图需要加载模型（10-15 秒），后续会快很多
- 换更小的模型：`ollama pull llava-phi3`

**更多问题？查看 [使用说明.md](./使用说明.md) 的”常见问题”章节**
